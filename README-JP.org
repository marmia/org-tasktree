* プロジェクト概要
Emacs 向けタスク管理プラグイン(org-tasktree)の開発

* org-tasktree とは

org-tasktree は、org-mode を編集 UI とし、SQLite を唯一のマスターデータベースとする
Emacs 用タスク管理プラグインです。

- org バッファは「編集ビュー」
- SQLite は「唯一の正（single source of truth）」
- タスクは任意の親子階層で管理されます
- domain / project / phase / group は分類用 tag として扱えます

* 特徴

- org-mode の標準機能のみを使用
  - org-todo / org-schedule / org-deadline / org-priority / org タグ
- SQLite ベースの高速な検索
- UID ベースの INSERT / UPDATE による安全な同期設計
- YAML による柔軟な検索クエリー定義
- node_tags による tag 検索

* 必要環境

- Emacs 29 以上（sqlite.el サポート必須）
- org-mode（Emacs 標準）
- 対応 OS:
  - macOS
  - Linux
- Windows:
  - v1 では動作保証外（将来対応検討）

* 初期設定

org-tasktree は初回起動時に、以下の設定を user-emacs-directory を基準に
自動的に初期化します。

- SQLite DB:
  - ~/.emacs.d/org-tasktree/tasktree.db
  - ~/.config/emacs/org-tasktree/tasktree.db など

- クエリーディレクトリ:
  - ~/.emacs.d/org-tasktree/queries/
  - ~/.config/emacs/org-tasktree/queries/

初回のみ、以下を実行してください。

#+begin_src elisp
M-x org-tasktree-init
#+end_src

これにより：

- DB ファイル作成
- テーブル作成
- クエリーディレクトリ作成

がすべて自動で行われます。

※ これらのパスは、後から defcustom で自由に変更可能です。

* 基本的な使い方

** タスクツリーを org-mode で一括作成して登録

1. 新規 org バッファで任意の親子階層を 1 から記述する。
2. ツリー全体を SQLite に同期する。
   - `org-tasktree-sync-buffer` で同期する。

** DB のタスクツリーを org-view にロードして編集

1. 既存 node を検索して org 形式の編集ビューを開く。
2. org-mode 上でツリーを編集する。
   - `uid` あり見出しは既存ノード更新、`uid` なし見出しは新規ノードとして扱う。
3. 編集した範囲のみを DB に再同期する。
   - `org-tasktree-sync-subtree` / `org-tasktree-sync-region` で同期する。

** 単体 node を UI で素早く登録

1. `M-x org-tasktree-find-node` を実行する。
2. find-file 風の階層補完で既存 node を辿り、最後に新規 node 名を入力する。
3. 開かれた編集バッファで必要項目（priority / scheduled / deadline / tags / content）を入力し、`C-c C-c` で確定する。
4. tag に domain / project / phase / group を設定すると、補完候補の表示色に反映される。

* 設計ポリシー

- nodes は UID をキーに INSERT / UPDATE します
- node_tags は node_id 単位で DELETE / INSERT により再生成します
- UID による一意性保証
- parent_id は org バッファには保持せず、org ツリーから再計算します
- org バッファの内容は常に「一時ビュー」
- DB の内容のみが正です
