#+TITLE: org-tasktree Specification - Commands
#+STARTUP: showall

* 初期化
** org-tasktree-init

*** 目的
- SQLite データベースの初期化処理を行う。

*** 処理内容

- 以下の処理を順番に実行する。

1. `org-tasktree-database-location` で指定されたパスを取得
2. 親ディレクトリが存在しない場合は作成
3. DB ファイルが存在しない場合は新規作成
4. `nodes` / `node_tags` / `meta` テーブルを作成
5. `meta` テーブルに初期データを登録
   - schema_version = "1"
   - created_at = 現在時刻（ISO 8601）
6. システム既定 Project「inbox」を `nodes` に登録
7. `org-tasktree-query-dir` が存在しない場合は作成

*** 実行タイミング

- ユーザーが明示的に実行した場合
- 他コマンド実行時に DB が存在しない場合は自動実行

*** 副作用

- DB ファイルおよびクエリディレクトリが作成される

* 登録/更新系
** org-tasktree-find-node
*** 目的
- 任意の node の更新/新規追加を行う。

*** 処理内容
1. nodes テーブルから の project / phase / group / task 情報を取得してミニバッファに表示する。
   - 検索条件: status='OPEN'
   - ミニバッファのプロンプト表示: `find node:`
   - ミニバッファのリスト表示:
     - 各 node をパス形式で表示する。(find コマンドのように'/'区切りで表示)
       - リスト表示例
         #+begin_example
         inbox/
         project1
         project1/phase1
         project1/phase1/group1
         project1/phase1/group1/task1
         project1/phase1/group1/task1/child1
         project2
         project2/phase2
         project2/phase2/group2
         project2/phase2/group2/task2
         project2/phase2/group2/task2/child2
         project2/phase2/group2/task2/child2/mago1
         #+end_example
     - 'node_type'ごとに色を分けて表示する。
       - Default 値: 
         - project: DeepSkyBlue
         - phase: MediumSeaGreen
         - group: DarkOrange
         - task: white

2. ユーザーが node を入力/選択して'Enter'キーを押すと node 編集バッファを開く
   - ユーザーが既存の node 名を入力した場合、更新処理として扱う。
   - ユーザーが新規の node 名を入力した場合、新規登録として扱う。
     - 新規 node は既存の node パスに続けて'/<node_name>'を追記して'Enter'キーを押す。
   - 注意点:
     - 新規 node 名に'/'を含めることはできない。('/'は常に path の区切り文字として解釈される)
       - 末尾につけられた'/'は削除して編集バッファ を表示
       - node 名の途中に'/'が記述されている場合はアラート表示
     - 複数階層の新規 node を一気に作成することはできない。
       - ex:
         - 既存 node パス: project1/phase1/group1
         - 新規 node: project1/phase1/group1/task1 -> OK
         - 新規 node: project1/phase1/group1/task1/child1 -> NG (task1/child1 を一気に作成することはできない)

3. 編集 バッファの表示/入力
   - 編集バッファは `widget.el` ベースとし、UI 上部に親 node のパス（project > phase > group）を表示する。
   - 入力項目
     - node_type
     - title
     - priority
     - scheduled
     - deadline
     - repeat
     - tasg
   - node_type 指定ルール
     - TOP 階層 node を更新/新規追加する場合は、'project'固定表示 (編集不可)
     - project 配下の node に指定できるのは、'phase/group/task'
     - phase 配下の node に指定できるのは、'group/task'
     - group 配下の node に指定できるのは、'group/task'
     - task 配下の node に指定できるのは、'task'
   - title フィールドにはミニバッファで入力/選択された node パスの末端の node 名を初期表示する。
   - scheduled / deadline は手入力と日付選択ダイアログの両方をサポートする。
     - `C-c C-s` で scheduled、`C-c C-d` で deadline の `org-read-date` を起動する。
     - `C-g` でキャンセルする。
   - `C-c C-c` 実行時に `scheduled < deadline` を検査する（どちらかが未指定なら検査しない）。
   - *重要* :
     - 初期カーソル位置は title の入力欄。
     - 既存ノードの UID は表示せず内部で保持する。

4. 更新/キャンセル
   - ユーザーが node 内容を編集し<C-c C-c> を押すとデータベースに編集内容を登録する。
   - <C-c C-k>を押すとキャンセルしてミニバッファ表示に戻る。

* 検索系
** 共通仕様
- status='OPEN' のタスクのみを対象とする。
- scheduled / deadline が NULL のタスクは除外する。
- 日付比較は `DATE(...)` で行い、日付文字列が無効な場合はエラーとして処理を中断し Messages に出力する。
- 並び順は project > phase > group > task の階層順で、同一階層内は title 昇順。
- repeat!=NULL の場合は、tasktree 展開時に'<SCHEDULE yyyy-mm-dd>'に`repeat`に設定されている値を付け足して出力する。
  - Ex: <SCHEDULED 2025-12-20 .+1d>

** org-tasktree-search-today-task
*** 目的
- scheduled が今日になっている task の tasktree を取得する。
*** 処理内容
- nodes テーブルから`scheduled=today and status=OPEN`の task と、そのタスクの親 node (project/phase/group)を取得する。
  - node 情報は重複なく正規化したデータを取得
  - 所得する属性情報: uid, todo_keyword, title, level, priority, scheduled, deadline, tags
- 出力専用バッファを開き、org-mode 形式で task tree を展開する
  - 出力専用バッファは`q`キーで閉じる
