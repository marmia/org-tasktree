#+TITLE: org-tasktree Specification - Sync and Commit
#+STARTUP: showall

* `TODO` キーワードと内部状態のマッピング仕様
** org 側 TODO キーワード

- org-mode 側で定義されている TODO キーワードをそのまま使用する。
- プラグイン側でキーワードの制約は行わない。
- `nodes.todo_keyword` には見出しの TODO キーワードをそのまま保存する。
- nil（キーワードなし）を許容する。

** 削除フラグ（DEL）

- `DEL` は TODO キーワードではなく、見出し先頭の削除フラグとして扱う。
- 形式: `* DEL TODO task1` のように `DEL` をアスタリスク直後に置く。
- `DEL` は `org-todo-keywords` に登録しない。

** 内部属性

- status: OPEN / DONE

** status の決定

- org-todo が done 状態 → status = DONE
- それ以外 → status = OPEN

** 新規ノード（UID 未設定）かつ done 状態の扱い

- UID を持たない見出しで、org-todo が done 状態の場合は完全に無視する
  - UID 採番しない
  - DB に INSERT しない

* コミット仕様（org → SQLite 同期）
** コミットスコープ方式

- 対象スコープ内のノードは `uid` をキーに INSERT/UPDATE する（nodes の DELETE/INSERT 再生成は行わない）。
- node_tags は node_id 単位で DELETE / INSERT により再生成してよい。
- スコープ外ノードは DB 上で一切変更しない

** create/update の境界

- `uid` が DB に存在する場合は update（UPDATE）として扱う。
- `uid` が DB に存在しない場合は user-error とする。
- UID 未設定の見出しのみ create（INSERT）として扱う。

** UPDATE 可能列 / 禁止列（nodes）

- 禁止（不変）:
  - id
  - uid
  - created_at
- 更新可能:
  - parent_id
  - todo_keyword / title
  - priority / scheduled / deadline / repeat / tags / content
  - status / closed_at
  - updated_at（UPDATE のたびに現在時刻）

** 親子関係の再構築

- parent_id は org ツリー構造から毎回再計算
- 親がスコープ外で特定できない場合は DB 値を維持する（keep）

** UID の付与

- UID 未設定の見出しは sync 時に自動で UID を付与する。

** content の抽出

- content は見出し直下の本文のみを対象とする。
- PROPERTIES などの drawer と SCHEDULED/DEADLINE 行は除外する。

** 削除（DEL）

- `DEL` フラグが付いた見出しは削除対象とする。
- 子孫ノードも再帰的に削除する。
- UID 未設定の `DEL` 見出しは無視する。

** 繰り返しタスクの扱い

- DB に保存する scheduled は、繰り返し指定を含まない「日付のみ」の値とする。
  例:
  - org:  SCHEDULED: <2025-12-11 Thu +1d>
  - DB:   scheduled = "2025-12-11"

- "+1d" などの繰り返しルール、および LAST_REPEAT や State 変更ログは DB に保存しない。
- 繰り返しタスクは、「常に現在の 1 インスタンス」としてのみ扱う。

** insert / update フロー

1. スコープ内ノードを収集（既存レコードの id / parent_id / created_at をキャッシュ）
2. 親子関係（parent_id）を org ツリーから再計算（親がスコープ外で org 側から特定できない場合はキャッシュ値を維持）
3. トランザクション開始
4. 親ノードから順に、`uid` が既存の場合は UPDATE、未登録の場合は INSERT（id/uid/created_at は不変、updated_at は現在時刻）
5. node_tags を対象 node_id について DELETE し、tags 文字列からパースしたタグを INSERT
6. COMMIT
