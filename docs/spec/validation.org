#+TITLE: org-tasktree Specification - Validation
#+STARTUP: showall

* バリデーション仕様
** nodes テーブル

*** 必須フィールド（NOT NULL チェック）

- uid
  - nil / 空文字は禁止。
  - UUID 形式であること（例: `00000000-0000-0000-0000-000000000000`）。
  - コミット対象スコープ内で重複しないこと（論理一意性）。

- title
  - nil / 空文字 / 空白のみは禁止。
  - 禁則文字: スラッシュ(/), 改行（\n/\r）, 制御コード（U+0000–U+001F, U+007F）

- status
  - 許容値:
    - "OPEN" / "DONE"
  - 上記以外の値はエラー。

- created_at / updated_at
  - nil でないこと。
  - 形式は ISO 8601 形式の文字列を基本とするが、厳密チェックは行わない。

*** 日付関連（scheduled / deadline / repeat）

- scheduled / deadline
  - いずれも NULL を許容する。
  - DB に保存する値は `YYYY-MM-DD` 形式であること（コミット時に正規化する）。
  - UI の入力は以下を許容する（手入力・日付選択ダイアログの両方をサポートする）。
    - `YYYY-MM-DD`
    - `YYYY/MM/DD`
    - `MM-DD`（ゼロ埋めは任意）
    - `MM/DD`（ゼロ埋めは任意）
    - `DD`（1 桁も可）
  - 省略形（`MM-DD` / `MM/DD` / `DD`）は「今日を含む未来のうち最も近い日付」に解決して `YYYY-MM-DD` として保存する。
    - 例: 今日が 2025/12/15 のとき、`1` は `2026-01-01` として保存する。
  - 両方が存在する場合（いずれも非 NULL）:
    - `scheduled <= deadline` を満たすこと。

- repeat
  - NULL を許容する。
  - org-mode の Repeat Syntax に沿った文字列のみ許可する。
  - Repeat Syntax
    - syntax: <+|++|.+><Number><d|w|m|y>[/<Number>]
    - <...> : 必須トークン（かつ | で選択肢）
    - [...] : 任意

*** priority

- priority
  - NULL を許容する。
  - NULL でない場合は、英数字 1 文字であること（例: `A` / `B` / `1`）。

*** tags

- tags
  - NULL を許容する。
  - 空文字は NULL とみなす。
  - 非 NULL の場合:
    - 入力は `:` 区切り（例: `tag1:tag2`）であること。
    - 各 tag に使える文字: 半角英数字 / `-` / `_`。
  - 正規化された実体は node_tags で管理する。
  - nodes.tags は入力値をそのまま保存する。

*** content

- content
  - NULL を許容する。
  - 複数行を許容する。
  - 空白のみの入力は NULL とみなす。

*** リレーション系フィールド

- parent_id
  - NULL を許容する（ルートノードなど）。
  - NULL でない場合:
    - 対応する id を持つノードが DB 内に存在すること。

- 外部キー制約
  - node_tags.node_id は nodes.id を参照する外部キーを付与する（ON DELETE CASCADE）。
  - parent_id には外部キーを付与しない。
  - DB 接続時に PRAGMA foreign_keys=ON を必須とする。

*** `DONE` / closed_at

- closed_at
  - NULL を許容する。
  - status = "DONE" のときにのみ値を持つことが望ましい。

** node_tags テーブル

*** 必須フィールド

- node_id
  - nil は禁止。
  - 整数であること。
  - 対応する id を持つノードが nodes テーブルに存在すること。

- tag
  - nil / 空文字は禁止。
  - 前後の空白はトリムして保存すること。

*** 一意性・整合性

- (node_id, tag) の組み合わせは、1 コミット内では重複しないこと。
- node_tags の内容は、nodes.tags から再生成できる形に保つことが望ましい。
