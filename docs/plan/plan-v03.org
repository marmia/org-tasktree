** DONE M32: 見出しに'/'を含む場合の対応
*** DONE 仕様の確認
- 現在の仕様では、`sync-*`で見出しに'/'を含む場合は、バリデーションチェックでエラーになる。
- 問題は、`sync-region/subtree`で更新対象の見出しと関係のない見出しに'/'を含まれている場合もバリデーションエラーとなる点である。
  
考えられる対応方法:
1. `title`の禁足文字から'/'を外す。
   そもそも'/'を禁足文字にしたのは、`find-node`のミニバッファで Node パスを'/'区切りで表示しているので title に'/'が入っていると紛らわしいからである。`title`に'/'を指定できないのは不便ではある。
   禁足文字から'/'を外す場合は、`find-node`のミニバッファの表示をどうするか検討する必要がある。
2. `sync-region/subtree`で更新対象外の見出しはバリデーションチェックしないようにする。
3. 1 と 2 の両方の修正を行う。

** M33 上位 Node の status が"DONE"になった場合に下位 Node も"DONE"に設定
*** DONE 仕様の確認
- `org-tasktree-sync-buffer`, `org-tasktree-sync-region`, `org-tasktree-sync-subtree` で Node の status が "DONE" になった場合に下位 Node も "DONE" に設定する。
- DB に既に存在する全子孫を一括で DONE にする。
- 更新対象は `nodes.status` / `nodes.todo_keyword` / `nodes.closed_at` / `nodes.updated_at` とする。
- 上位 Node と子孫 Node の `todo_keyword` は "DONE" に設定する。
- 上位 Node と子孫 Node の `closed_at` は sync 実行時の timestamp に設定する。
- 上位 Node が "DONE" になった場合、sync 対象スコープ外の DB 子孫も DONE にする。
- 上位 Node 自身が新規作成された場合は、DB 上の既存子孫が存在しないため追加更新は発生しない。
- 上位 Node が "OPEN" の場合、子孫の status は変更しない。
