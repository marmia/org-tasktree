* プロジェクト概要
Emacs 向けタスク管理プラグイン(org-tasktree)の開発

* org-tasktree とは

org-tasktree は、org-mode を編集 UI とし、SQLite を唯一のマスターデータベースとする  
Emacs 用タスク管理プラグインです。

- org バッファは「編集ビュー」
- SQLite は「唯一の正（single source of truth）」
- タスクは階層構造（Project / Phase / Group / Task）で管理されます

* 特徴

- org-mode の標準機能のみを使用
  - org-todo / org-schedule / org-deadline / org-priority / org タグ
- SQLite ベースの高速な検索
- delete / insert 方式による安全な同期設計
- YAML による柔軟な検索クエリー定義
- Project / Phase 未指定タスクは自動的に inbox に集約

* 必要環境

- Emacs 29 以上（sqlite サポート前提）
- org-mode（Emacs 標準）
- Vertico（任意・推奨）

* 初期設定

org-tasktree は初回起動時に、以下の設定を user-emacs-directory を基準に
自動的に初期化します。

- SQLite DB:
  - ~/.emacs.d/org-tasktree/tasktree.db
  - ~/.config/emacs/org-tasktree/tasktree.db など

- クエリーディレクトリ:
  - ~/.emacs.d/org-tasktree/queries/
  - ~/.config/emacs/org-tasktree/queries/

初回のみ、以下を実行してください。

#+begin_src elisp
M-x org-tasktree-init
#+end_src

これにより：

- DB ファイル作成
- テーブル作成
- inbox Project 作成
- クエリーディレクトリ作成

がすべて自動で行われます。

※ これらのパスは、後から defcustom で自由に変更可能です。

* 基本的な使い方

** タスクの登録

org-mode で通常どおりタスクツリーを記述します。

#+begin_src org
 * PROJ サンプルプロジェクト
 ** PHASE 設計
 *** TODO 画面構成を考える :ui:design:
SCHEDULED: <2025-12-10 Tue>
#+end_src

次に以下を実行します。

#+begin_src elisp
M-x org-tasktree-sync-buffer
#+end_src

** 部分同期

- リージョンのみ同期:
  - `M-x org-tasktree-sync-region`
- カーソル位置のサブツリーのみ同期:
  - `M-x org-tasktree-sync-subtree`

* 検索の使い方

** クエリー作成

#+begin_src elisp
M-x org-tasktree-new-query
#+end_src

YAML 形式で検索条件を記述します。

#+begin_src yaml
name: today
scheduled:
  from: 2025-12-09
  to: 2025-12-09
status: open
#+end_src

** クエリー保存

#+begin_src elisp
M-x org-tasktree-save-query
#+end_src

** クエリー呼び出し

#+begin_src elisp
M-x org-tasktree-find-query
#+end_src

** 検索実行（v1）

#+begin_src elisp
M-x org-tasktree-search-to-buffer
#+end_src

検索結果は org 形式の専用バッファに表示されます。

* 設計ポリシー

- UPDATE 文は一切使いません（DELETE / INSERT のみ）
- UID による一意性保証
- org バッファの内容は常に「一時ビュー」
- DB の内容のみが正です

* v2 以降の予定

- 検索結果のカーソル位置直接挿入
- ガントチャートビュー
- DB マイグレーション機構
- CLOSED 表示と変更履歴管理
- XDG 対応
