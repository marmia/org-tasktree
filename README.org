* プロジェクト概要
Emacs 向けタスク管理プラグイン(org-tasktree)の開発

* org-tasktree とは

org-tasktree は、org-mode を編集 UI とし、SQLite を唯一のマスターデータベースとする  
Emacs 用タスク管理プラグインです。

- org バッファは「編集ビュー」
- SQLite は「唯一の正（single source of truth）」
- タスクは階層構造（Project / Phase / Group / Task）で管理されます

* 特徴

- org-mode の標準機能のみを使用
  - org-todo / org-schedule / org-deadline / org-priority / org タグ
- SQLite ベースの高速な検索
- delete / insert 方式による安全な同期設計
- YAML による柔軟な検索クエリー定義
- Project / Phase 未指定タスクは自動的に inbox に集約

* 必要環境

- Emacs 29 以上（sqlite.el サポート必須）
- org-mode（Emacs 標準）
- 対応 OS:
  - macOS
  - Linux
- Windows:
  - v1 では動作保証外（将来対応検討）

* 初期設定

org-tasktree は初回起動時に、以下の設定を user-emacs-directory を基準に
自動的に初期化します。

- SQLite DB:
  - ~/.emacs.d/org-tasktree/tasktree.db
  - ~/.config/emacs/org-tasktree/tasktree.db など

- クエリーディレクトリ:
  - ~/.emacs.d/org-tasktree/queries/
  - ~/.config/emacs/org-tasktree/queries/

初回のみ、以下を実行してください。

#+begin_src elisp
M-x org-tasktree-init
#+end_src

これにより：

- DB ファイル作成
- テーブル作成
- inbox Project 作成
- クエリーディレクトリ作成

がすべて自動で行われます。

※ これらのパスは、後から defcustom で自由に変更可能です。

* ユースケース

** タスクツリーを org-mode で一括作成して登録（予定）

1. 新規 org バッファで Project / Phase / Group / Task のツリーを 1 から記述する。
2. ツリー全体を SQLite に同期する。
   - v0.2 以降で `org-tasktree-sync-buffer` を提供予定。

** DB のタスクツリーを org-view にロードして編集（予定）

1. 既存 Project/Phase を検索して org 形式の編集ビューを開く。
2. org-mode 上でツリーを編集する。
   - `uid` あり見出しは既存ノード更新、`uid` なし見出しは新規ノードとして扱う。
3. 編集した範囲のみを DB に再同期する。
   - v0.2 以降で `org-tasktree-sync-subtree` / `org-tasktree-sync-region` を提供予定。

** 単体タスクを UI で素早く登録（v0.1）

1. `M-x org-tasktree-find-task` を実行する。
2. find-file 風の階層補完で Project / Phase / Group を辿り、最後に task 名を入力する。
3. 開かれた編集バッファで必要項目（priority / scheduled / deadline / tags）を入力し、`C-c C-c` で確定する。
4. Project 未指定の task は自動的に Inbox Project に収容される。

** Inbox に集約して後で整理

1. とりあえず単体タスクを Inbox に登録する（上記手順）。
2. まとまった時間で Inbox を開き、適切な Project/Phase に移動・整理する。

* 設計ポリシー

- UPDATE 文は一切使いません（DELETE / INSERT のみ）
- UID による一意性保証
- org バッファの内容は常に「一時ビュー」
- DB の内容のみが正です
