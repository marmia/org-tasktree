* Project Overview

Development of =org-tasktree=, a task management plugin for Emacs.

* What is org-tasktree?

=org-tasktree= is a task management plugin for Emacs that uses =org-mode= as
the editing UI and SQLite as the single master database.

- Org buffers are editing views.
- SQLite is the single source of truth.
- Tasks are managed in a hierarchical structure: Project / Phase / Group / Task.

* Features

- Uses only standard =org-mode= features:
  - =org-todo= / =org-schedule= / =org-deadline= / =org-priority= / Org tags
- Fast SQLite-based search
- Safe synchronization design based on delete / insert operations
- Flexible search query definitions using YAML
- Tasks without a Project / Phase are automatically collected in the inbox

* Requirements

- Emacs 29 or later, with =sqlite.el= support
- =org-mode=, bundled with Emacs
- Supported OS:
  - macOS
  - Linux
- Windows:
  - Not officially supported in v1; future support is under consideration

* Initial Setup

On first startup, =org-tasktree= automatically initializes the following
settings relative to =user-emacs-directory=.

- SQLite DB:
  - =~/.emacs.d/org-tasktree/tasktree.db=
  - =~/.config/emacs/org-tasktree/tasktree.db=, etc.

- Query directory:
  - =~/.emacs.d/org-tasktree/queries/=
  - =~/.config/emacs/org-tasktree/queries/=

Run the following command once on first use.

#+begin_src elisp
M-x org-tasktree-init
#+end_src

This automatically performs all of the following:

- Creates the DB file
- Creates the tables
- Creates the Inbox Project
- Creates the query directory

These paths can be freely changed later via =defcustom=.

* Use Cases

** Create and Register a Task Tree in org-mode (Planned)

1. Write a Project / Phase / Group / Task tree from scratch in a new Org buffer.
2. Synchronize the entire tree to SQLite.
   - =org-tasktree-sync-buffer= is planned for v0.2 or later.

** Load a DB Task Tree into an Org View and Edit It (Planned)

1. Search for an existing Project/Phase and open an Org-format editing view.
2. Edit the tree in =org-mode=.
   - Headings with a =uid= update existing nodes; headings without a =uid= are
     treated as new nodes.
3. Resynchronize only the edited range to the DB.
   - =org-tasktree-sync-subtree= / =org-tasktree-sync-region= are planned for
     v0.2 or later.

** Quickly Register a Single Task from the UI

1. Run =M-x org-tasktree-find-node=.
2. Follow Project / Phase / Group via find-file-style hierarchical completion,
   then enter the task name at the end.
3. In the opened editing buffer, enter the required fields: priority /
   scheduled / deadline / tags / content. Confirm with =C-c C-c=.
4. Tasks without a Project are automatically placed in the Inbox Project.

** Collect Tasks in Inbox and Organize Later

1. First, register standalone tasks in Inbox using the steps above.
2. Later, open Inbox during focused organization time and move tasks to the
   appropriate Project/Phase.

* Design Policy

- UPDATE statements are never used; only DELETE / INSERT operations are used.
- Uniqueness is guaranteed by UID.
- Org buffer contents are always temporary views.
- Only DB contents are authoritative.
